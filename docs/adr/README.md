# Architecture Decision Records

ADRs record important decisions and why they were made. They should be short, reviewable, and linked from architecture docs when relevant.

## When to add an ADR

Add an ADR for decisions that affect architecture, package boundaries, provider strategy, data persistence, API contracts, security posture, or long-lived workflow conventions.

Do not add an ADR for every small implementation change.
