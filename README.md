# TrustTrace

## Development Workflow

- **No direct pushes to `main`.** All changes must go through a Pull Request (PR).
- **Use feature branches:** `feat/...`, `fix/...`, `chore/...` (branch off `main`).
- **PR requirements to merge:**
  - At least **1 approval**
  - **All CI checks must pass**
- **Merge method:** **Squash and merge**. Delete the branch after merging.
